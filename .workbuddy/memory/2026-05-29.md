# 2026-05-29 工作日志

## 飞书机器人项目搭建

创建了 `FeishuCardBot` 项目，实现飞书 WebSocket 长连接机器人，用于通过消息卡片创建 PLM 部件。

### 项目结构
```
FeishuCardBot/
├── src/
│   ├── index.js          # WebSocket 长连接入口，消息和卡片回调处理
│   ├── card-template.js  # 消息卡片 JSON 模板（表单：物料名称/项目号/所在库/视图/文件夹/分类）
│   └── plm-client.js     # PLM 系统接口调用模块
├── package.json          # 依赖：@larksuiteoapi/node-sdk, axios, dotenv
├── .env.example          # 环境变量模板
└── README.md             # 使用文档
```

### 核心功能
1. **WebSocket 长连接**：无需公网 IP，使用 `WS.Client` 连接 `wss://ws.feishu.cn/skylark`
2. **消息卡片表单**：包含 1 个输入框（物料名称）+ 5 个下拉选择（项目号/所在库/视图/文件夹/分类）
3. **卡片回调处理**：监听 `card.form.submit` 事件，提取表单数据调用 PLM 接口
4. **PLM 接口封装**：`plm-client.js` 提供创建部件和获取下拉选项的方法（示例实现，需对接真实接口）

### 启动方式
```bash
cp .env.example .env  # 配置 FEISHU_APP_ID/SECRET 和 PLM_API_BASE_URL/TOKEN
npm install
npm start
```

### 关键修复（ESM → CommonJS）
飞书 SDK `@larksuiteoapi/node-sdk` 是 CommonJS 模块，不支持 ESM 命名导入。
修复方案：
1. 移除 `package.json` 中的 `"type": "module"`
2. 所有文件改用 `require()` / `module.exports`
3. SDK 导入方式：`const { Client, WSClient, EventDispatcher, CardActionHandler } = require('@larksuiteoapi/node-sdk')`

### SDK 使用模式
```javascript
// 1. 创建事件分发器
const eventDispatcher = new EventDispatcher().register({
  'im.message.receive_v1': async (data) => { /* 处理消息 */ }
})

// 2. 创建卡片回调处理器
const cardHandler = new CardActionHandler({}, async (data) => { /* 处理卡片回调 */ })

// 3. 创建 WSClient 并启动
const wsClient = new WSClient({ appId, appSecret, domain: 0, onReady, onError })
wsClient.start({ eventDispatcher, cardHandler })
```

### 飞书开放平台配置要点
1. 事件与回调 → 订阅方式 → 选择「使用长连接接收事件/回调」
2. 添加事件订阅：`im.message.receive_v1`
3. 卡片回调 → 同样选择「使用长连接接收卡片回调」
4. 权限：`im:message`, `im:message:send_as_bot`, `im:chat`, `im:resource`
5. 发布版本后才能正常连接

### 关键修复2：卡片只显示标题问题
- 飞书 `form`/`input`/`select_static` 组件需要卡片 **2.0** 格式
- 必须加 `"schema": "2.0"`，内容放在 `body.elements` 而非顶层 `elements`
- 卡片 2.0 回调中表单数据在 `action.form_value`（1.0 在 `data.form_value`）
- 提交按钮识别通过 `action.name === 'submit_btn'`
